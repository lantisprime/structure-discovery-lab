# eq_eval_set_v1 — blind pack (for the lab)

You are given 12 univariate daily time series (`datasets/unit_01.csv` …
`unit_12.csv`, columns `date,value`, n=400 each). Units and physical meaning
are intentionally unspecified. Some contain recoverable deterministic laws,
some contain structure without any equation, some are pure noise, and some
are traps. The mixture proportions are sealed.

## Your task

Run the lab's normal Phase-5 equation-discovery cycle per unit using the v4
machinery (`src/eq_selection_v4.py`) and the playbook contract (Q-1..Q-8):

1. Detection gate first (Q-1): fit nothing the detection layer calls NULL —
   a refusal is a first-class, scoreable answer.
2. Registered candidate-family menu, chronological 60/20/20 split, pure
   held-out selection (`select_family`), null-equation generator with
   B ≥ `b_required(corrected_alpha(0.05, m))`, verdict-time correction
   in-run (`adjudicate`) — no deferral language.
3. Residual scan (`residual_scan_mc`): run once with an empty whitelist;
   if you attribute a line, run again with it whitelisted and report both.
4. Report frequency uncertainty via `profile_ci_omega`.
5. Stationarity/era checks apply (a law that holds in only part of a series
   must be reported as such, not pooled).
6. For every FAILED verdict on a series you believe structured, include the
   power statement (smallest amplitude your design could have seen).

## Output contract

One JSON: `results/eq_eval_verdicts.json` —
`{unit_id: {verdict, periods_d, period_cis, residual_scan: {no_whitelist_p,
whitelisted_p, whitelisted_lines}, era_flag, power_floor_amp, notes}}`.
Hash-commit it BEFORE unsealing `answer_key/`. Score with
`src/score_eq_eval.py`.

## Multiplicity

Declare the family `eq.eval-set-v1` with m = number of units on which you
run a fit (refusals charge 0). The verdict-time correction uses the ratified
convention v1 (Šidák .05, cumulative family m).

## Role separation

The pack was generated by the same agent lineage that built the v4
machinery. To keep the benchmark blind in practice, the run must be executed
by an agent/session that has NOT read `answer_key/`, `src/generate_eq_eval.py`
or `manifest.json` — see SEAL_NOTICE.md.
