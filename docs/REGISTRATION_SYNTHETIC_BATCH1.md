# Pre-Registration — Synthetic Batch 1 ("Biased Lottery Chamber", Case Study 2)

Registered: 2026-06-12, before any detection run on generated outcome data (A6).
Analyst: Claude (with Cha). Status: **CALIBRATION / EXPLORATION** — this batch makes
no claim about any real lottery. Ground truth is planted; the deliverables are
equations and power maps, not detections.

## Motivation

Case Study 1 established that the pipeline separates a null draw process from
structured covariates on real data. This batch inverts the question: with known
planted bias, how large, persistent, and long-lived must a bias be before the
existing instruments detect it (Experiment A), survive confirmation (Experiment C),
and which instrument sees which mechanism (Experiment D)?

Registered targets (per `uploads/01_synthetic_biased_lottery_chamber.md` case card):

1. **Detectability frontier** — n_min(δ, α, β) ≈ (z₁₋α + z₁₋β)² / (δᵀ Σ₀⁺ δ),
   validated against empirical detection probability.
2. **Era half-life threshold** — minimum decay constant τ for a signal that fires
   in exploration to survive the registered confirmation protocol.
3. **Instrument power map** — chi-square vs lasso support vs graphon spectral norm
   as bias mass spreads across balls.

**Deferred (not in this batch):** Experiment B (dead transient / hard cutoff). It is
a degenerate case of Experiment C (τ → era length, then 0) and will be registered
separately if C's results leave it informative.

## Datasets

- **Source:** generated by `src/domains/synthetic_lottery.py` (this batch's domain
  module). Pool P = 55, k = 6 — matches Grand Lotto 6/55 so frontiers are directly
  comparable to Case Study 1 bounds.
- **Null simulator (A1):** i.i.d. uniform 6-without-replacement via
  `core.discrete_draws.fast_draws`. The fair mode of the synthetic generator is
  provably identical (Gumbel-key argsort with equal weights = uniform permutation).
- **Exploration / confirmation split:** for each replicate, the first n_e draws are
  exploration; the next n_c draws are confirmation, generated by *continuing the
  same bias process* (decay clock keeps running). Confirmation data is never seen
  by the detection step. n_e and n_c are registered per experiment below.
- **Seeds:** master seed 20260612; per-cell seeds derived as
  `seed = master * 10^6 + cell_index`, recorded in the results JSON.

## Bias injection (registered mechanism)

Weighted sampling without replacement via Gumbel top-k with weights w_i(t).
Nominal bias δ maps to weights at first order (w_j = p_j / (6/P)); because the
mapping is not exact for without-replacement sampling, the **realized** inclusion
deviation δ̂ is measured by MC (K = 200,000 draws per spec) and δ̂ — not nominal
δ — is what enters the frontier equation. Realized δ̂ per cell is reported.

Modes registered: single-ball excess (mass-conserving), multi-ball spread,
exponential amplitude decay w_j(t) − 1 ∝ e^{−t/τ}.

## Instruments, statistics, thresholds

All instruments already admitted (KB cards; no new onboarding). Per-cell detection
decision uses MC null calibration, K = 2000 null replicates, +1-corrected p
(`stats.p_perm`), per A1/A2.

- **I1 Per-ball chi-square** (`core.discrete_draws.chi2_counts`) — primary
  frequency instrument; the frontier equation is stated for this statistic.
- **I2 Lasso sparse-bias support** (`concentration_exclusion.py` A2 pattern) —
  support size > 0, λ calibrated to ≥ 99% empty support on null.
- **I3 Graphon co-occurrence spectral norm** (`graphon_cooccurrence.py` B1) —
  candidate winner for spread/cluster bias (Experiment D).
- **I4 Rolling-window persistence** (cross-window deviation correlation,
  `half_deviation_corr` generalization) — Experiment C only.

Family per cell: m = 3 counted tests (I1, I2, I3; I4 enters only C where it is
the registered statistic of interest, m = 4 there). Per-test threshold:
Šidák α' = `stats.sidak(0.05, m)`. Detection = any counted test below α'.

## Experiments

### A — Detectability frontier (priority 1)

Single hot ball j = 17 (arbitrary, fixed). Relative excess
r = ε / (6/P) ∈ {0.05, 0.10, 0.20, 0.40, 0.80}; n ∈ {100, 200, 400, 800, 1600, 3200};
R = 200 replicates per cell. Output: empirical detection probability per cell vs
theoretical n_min at α' and power 1−β = 0.80.

**Registered prediction:** empirical n_min tracks the theoretical frontier within a
factor of 2 (multiplicity and without-replacement covariance push it up); ratio
reported per r.

### C — Era half-life (priority 2)

δ₀ fixed at the smallest r from A that is detectable at n_e = 400 with power ≥ 0.8.
τ ∈ {25, 50, 100, 200, 400, 800, ∞} draws; n_e = 400, n_c = 200; R = 200.
Protocol per replicate: detect on exploration (as in A); if fired, re-test the
*same registered statistic* on confirmation draws only at α = 0.05 (single test —
family was fixed by the exploration hit, per A6). Output: confirmation survival
rate vs τ; **τ_min** = smallest τ with survival ≥ 0.8 among true detections.

**Registered prediction:** survival collapses when τ ≲ n_e (most signal mass spent
before the confirmation window); τ_min expected in the n_e–2n_e range.

### D — Instrument power map (priority 3)

Total bias mass fixed at the c = 1 mass of A's r = 0.40 cell, spread evenly over
c ∈ {1, 2, 4, 8, 16} balls (fixed ball sets, listed in the script). n = 800,
R = 200. Output: per-instrument (I1, I2, I3) detection probability vs c.

**Registered prediction:** I1/I2 dominate at small c; I3 degrades more slowly as c
grows but no instrument retains power ≥ 0.5 at c = 16 at this mass.

## Multiplicity (registered before running)

This is a power study: each cell's detection rule (m = 3 or 4, Šidák) is the unit
being measured, so cells are not a testing family — no cross-cell correction.
The only confirmatory-style claims are the three registered predictions above;
they are descriptive comparisons to theory, not p-valued hypotheses.

## Decision layer (kept separate, A7)

Actionability gate A(D) = Pr(survival | detection) × edge − cost is computed from
A + C outputs as a worked example with PCSO-like cost parameters, clearly labeled
synthetic. Structure ⇏ edge; edge ⇏ actionable edge.
